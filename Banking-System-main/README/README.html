Project Documentation

Overview

This project consists of a simple banking application that allows users to log in, view their checking and savings account balances, make deposits, transfer funds, and explore a placeholder page for future crypto investments. The project is built using HTML, CSS, and JavaScript, with Firebase as the backend database.

File Structure

index.html

The index.html file is the login and registration page for the application. Users can enter their username and password to log in or register for a new account. Upon successful login, users are redirected to the welcome.html page.

welcome.html

The welcome.html file is the main page that displays the user's financial information. It shows checking and savings account balances, provides buttons to make deposits and transfer funds, and offers a link to the crypto.html page. All the buttons are designed to toggle visibility of their respective input fields.

crypto.html

The crypto.html file is a placeholder page for future implementation of crypto investments. The contents of this page should be discussed with the professor or other students to fulfill the crypto investment requirement in the assignment document.

login.js

The login.js file handles user authentication and registration. It interacts with the Firebase database to store user information, such as username and password. It is important to note that you need to change the location of the JSON file in the argument of the const serviceAccount = require() function on line 8 to match your local machine.

bank_account_script.js

The bank_account_script.js file contains the functionality for updating the checking and savings balances displayed on the welcome.html page. It interacts with the Firebase database to store and retrieve the user's financial information in real-time.

Firebase Database Setup

The Firebase database is set up with two collections: users and financial. The users collection stores the username and password of each registered user. The financial collection stores the checking and savings account balances of each user, with the document ID matching the corresponding user document ID.

Testing Functionality

To test the functionality of the website from start to finish, follow these steps:

  1. Start the application by running login.js.
  2. Open a browser and navigate to the login page (usually at localhost:4000).
  3. Register a new user or log in with an existing account.
  4. Once logged in, you will be redirected to the welcome.html page.
  5. Test the deposit, transfer funds, and crypto investment
  6. Check the Firebase database to ensure that the financial information is being updated in real-time.
